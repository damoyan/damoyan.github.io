Swift学习笔记

可选类型(Optional)

可选(Optional)类型, 表示”那儿有个值, 并且它等于x”或者”那儿没有值”, 它有点像nil, 但是可以用在任何类型上, 比nil更加安全也更有表现力.

注意:

C 和 Objective-C 中并没有可选类型这个概念。最接近的是 Objective-C 中的一个特性，一个方法要不返回一个对象要不返回nil，nil表示“缺少一个合法的对象”。然而，这只对对象起作用——对于结构体，基本的 C 类型或者枚举类型不起作用。对于这些类型，Objective-C 方法一般会返回一个特殊值（比如NSNotFound）来暗示值缺失。这种方法假设方法的调用者知道并记得对特殊值进行判断。然而，Swift 的可选类型可以让你暗示任意类型的值缺失，并不需要一个特殊值。

nil

nil不可以用于非可选的常量和变量.

注意

Swift 的nil和 Objective-C 中的nil并不一样。在 Objective-C 中，nil是一个指向不存在对象的指针。在 Swift 中，nil不是指针——它是一个确定的值，用来表示值缺失。任何类型的可选状态都可以被设置为nil，不只是对象类型。

隐式解析可选类型

有时候在程序架构中，第一次被赋值之后，可以确定一个可选类型总会有值。在这种情况下，每次都要判断和解析可选值是非常低效的，因为可以确定它总会有值。

这种类型的可选状态被定义为隐式解析可选类型（implicitly unwrapped optionals）。把想要用作可选的类型的后面的问号（String?）改成感叹号（String!）来声明一个隐式解析可选类型。

-

类型安全, 会在编译时进行类型检查的.

类型推断, 浮点数默认推断为Double类型.

-

变量和常量的命名可以使用Unicode字符. 但是不能包含数学符号, 剪头, 保留的(或者非法的)Unicode码位, 连线与制表符. 也不能以数字开头.

输出函数:println和print, 是全局的.

注释和C语言相似 //表示单行注释, /**/用于多行注释, 多行注释可以嵌套, 比如/* /**/ */这种也是可以的, 这个可以用来注释大段代码, 哪怕代码中以及有了多行注释.

每行末尾的分号不是必须的

-

Swift所有类型都是首字母大写的. 包括基本类型.

Swift提供了8, 16, 32, 64位的有符号和无符号整数类型. 命名方式举例: 8位无符号整数类型是UInt8, 32位有符号整数类型是Int32. 可以通过min和max访问对应类型的最小值和最大值: UInt8.min是UInt8的最小值. Int和UInt类型的长度和当前平台的原生字长相同.

浮点数: Double表示64位浮点数, 当你需要存储很大或者精度很高的浮点数时使用; Float表示32位浮点数.

数值型字面量(字面量的意思是直接的数值, 不是变量保存的)

  • 十进制数, 没有前缀
  • 二进制数, 前缀0b
  • 八进制数, 前缀0o
  • 十六进制数, 前缀0x
